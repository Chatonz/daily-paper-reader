## 摘要
世界动作模型（WAMs）通过基于视频的世界建模作为机器人动作学习的密集监督，为视觉-语言-动作（VLA）策略提供了一种有前景的替代方案。现有的WAMs在物理接地执行方面表现出色，但通常缺乏基于VLM的VLA中用于分解粗略指令的显式语言级规划接口。当家务任务涉及复杂的多步目标时，这种分解变得重要，因为粗略的用户命令需要转换为一系列细粒度的可执行子任务。同时，该领域仍缺乏VLA和WAM执行能力之间公平的实体机器人比较，因为现有系统在数据、机器人具身和任务协议上往往不同。为了解决分解差距以及需要受控的WAM-VLA比较，我们引入了DSWAM，一种用于细粒度机器人操作的双系统世界动作基础模型。DSWAM保留系统1 WAM执行器作为默认控制路径，并仅在任务分解有用时可选地激活系统2视觉-语言子任务规划器。规划器从短期视觉历史和全局任务提示中预测可执行子任务，而WAM执行器对每个指令或子任务执行具有世界感知的动作生成。执行器通过动作预测和视频联合训练进行训练，但推理时直接预测动作块，无需显式的未来视频生成。为了使该执行路径在实体机器人上实用，我们进一步集成了TensorRT加速、异步执行和实时分块（RTC），以便策略查询不会阻塞机器人控制。为了与VLA策略进行公平的实体机器人比较，我们在DeMaVLA真实世界可变形操作设置下构建并评估DSWAM，匹配了机器人平台、预训练数据、后训练数据和评估标准。

## Abstract
World Action Models (WAMs) provide a promising alternative to Vision-Language-Action (VLA) policies by using video-based world modeling as dense supervision for robot action learning. Existing WAMs excel at physically grounded execution, but typically lack the explicit language-level planning interface in VLM-based VLAs for decomposing coarse instructions. Such decomposition becomes important when household tasks involve complex multi-step goals, where coarse user commands need to be converted into sequences of fine-grained executable subtasks. Meanwhile, the field still lacks a fair real-robot comparison between VLA and WAM execution capabilities, since existing systems often differ in data, robot embodiments, and task protocols. To address both the decomposition gap and the need for a controlled WAM-VLA comparison, we introduce DSWAM, a Dual-System World Action Foundation Model for fine-grained robot manipulation. DSWAM keeps a System 1 WAM executor as the default control path and optionally activates a System 2 vision-language subtask planner only when task decomposition is useful. The planner predicts executable subtasks from short-term visual history and a global task prompt, while the WAM executor performs world-aware action generation for each instruction or subtask. The executor is trained with action prediction and video co-training, but inference directly predicts action chunks without explicit future video generation. To make this execution path practical on real robots, we further integrate TensorRT acceleration, asynchronous execution, and real-time chunking (RTC) so that policy queries do not block robot control. To provide a fair real-robot comparison with VLA policies, we build and evaluate DSWAM under the DeMaVLA real-world deformable manipulation setting with matched robot platform, pretraining data, post-training data, and evaluation criteria.

- **问题**：现有世界动作模型（WAMs）在物理接地执行上表现优异，但缺乏像 VLM-based VLA 那样的显式语言级规划接口，无法将粗粒度用户指令分解为细粒度可执行子任务。同时，VLA 策略的“观测-动作”直接映射缺乏对场景物理动态建模的密集监督。此外，由于数据来源、机器人具身、任务协议不统一，WAM 与 VLA 之间缺乏公平的实体比较。
- **动机**：需要一种架构，既能保留 WAM 的物理执行优势，又能按需引入高层规划，并在同一实验条件下与 VLA 进行公平对比。
- **整体含义**：DSWAM 通过双系统设计（System 1 WAM 执行器 + System 2 可选规划器），桥接了 WAM 与 VLA 的差距，首次在匹配的 DeMaVLA 设置下证实 WAM 执行器在精细操作上的优势，为机器人基础策略提供了可扩展的设计范式。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：语义任务分解与物理执行解耦。System 1 作为默认控制路径，负责世界感知动作生成；System 2 为可选的高层规划器，仅在需要分解粗指令时激活，输出有序子任务序列。
- **关键技术细节**：
  - **System 1 – WAM 执行器**：
    - 输入：多视图 RGB 图像 \(I_{1,2,3}^t\)、语言指令或子任务 \(\ell_t\)、本体状态 \(q_t\)。
    - 编码为潜在世界特征 \(z_t = f_\theta(I_{1,2,3}^t, \ell_t, q_t)\)，直接预测动作块 \(A_t = [a_t, \dots, a_{t+H-1}]\)。
    - 训练：动作预测损失 \(\mathcal{L}_{\text{act}}\) + 视频协同训练损失 \(\mathcal{L}_{\text{vid}}\)（共享流匹配公式），未来视觉 token 仅提供监督，不用于推理。
    - 推理：直接输出动作块，不生成未来视频。
    - 部署优化：TensorRT 加速（拆分为视觉上下文引擎和动作去噪引擎）、异步执行、实时分块（RTC）。
  - **System 2 – 视觉-语言子任务规划器**：
    - 基于 Rynnbrain4B 风格的 VLM，输入短期视觉历史（最近5帧，1Hz采样）、全局任务提示 \(p\)，输出当前子任务指令 \(s_t = P_\phi(X_t, p)\)。
    - 训练数据：从机器人轨迹视频中分割出子任务区间，构造转换感知标签（内部/边界/结束），优化自回归语言建模损失 \(\mathcal{L}_{\text{plan}}\)。
    - 运行时：每 \(\Delta t=2\)s 更新一次子任务。
    - 可选性：不使用 System 2 时，原始指令直接作为 System 1 的语言条件。
- **公式/算法流程**：
  - 训练目标：\(\mathcal{L} = \mathcal{L}_{\text{act}} + \lambda_{\text{vid}} \mathcal{L}_{\text{vid}}\)，其中 \(\mathcal{L}_{\text{FM}} (y) = \mathbb{E}_{y,\epsilon,\tau} [\| v_\theta(y_\tau, \tau, O_t) - (\epsilon - y) \|_2^2]\)。

## 6. 主要结论与发现

- **仿真**：DSWAM 在 RoboTwin 2.0 上达到 Clean 92.38%、Randomized 91.90%，优于所有基线（包括 VLA 和 WAM）。
- **真实折叠**：在 WAM-only 模式下，DSWAM 成功率达 96.3%，平均耗时 1′44″，相比 DeMaVLA（92.5%, 2′18″）显著提升；相比 π0（76.3%, 2′26″）更大优势。
- **System 2**：子任务监督可将粗指令成功率从 75.7% 提升至 100%，每次执行失误从 3.53 降至 0.65。
- **效率**：TensorRT BF16 将延迟从 198.2 ms 降至 73.8 ms（2.69×加速），异步 RTC 进一步缩短完成时间并提升成功率。
- **结论**：DSWAM 有效结合 WAM 的物理执行与 VLM 的规划能力，在匹配条件下证明了 WAM 执行器在细粒度操作上的优势，且高效的部署使其可实际应用。

## 7. 优点

- **方法创新**：首次提出 WAM + 可选 VLM 规划器的双系统架构，实现语义与物理解耦，兼顾原子指令和复杂多步任务。
- **公平比较**：通过 DeMaVLA 匹配设置，消除了数据、具身、协议差异，提供了 WAM 与 VLA 执行力的可信对比。
- **工程实用**：TensorRT 加速、RTC、异步执行使大型模型可用于实时机器人控制；视频协同训练仅在训练期起作用，推理零开销。
- **实验充分**：覆盖仿真大规模 benchmark、真实精细操作、子任务消融、部署效率多个维度。

## 8. 不足与局限

- **实验覆盖局限**：真实任务仅评估折叠和简单排序，未涉及更复杂的多步骤家庭任务（如做饭、整理多个物体）；仿真任务虽多但可能侧重某些类型。
